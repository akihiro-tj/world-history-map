  Question 3/5: 年代選択UIの形式

  年代選択のUIがどのような形式であるべきか未定義です。ユーザーがどのように年
  代を選ぶかは操作性に大きく影響します。

  Recommended: Option B - スライダーは時間の連続性を直感的に表現でき、「縦の
  つながり」を意識しやすいUIです。ドラッグで年代を変えながら地図の変化を追え
  るため、学習体験として優れています。

  | Option | Description           |
  |--------|-----------------------|
  | A      | ドロップダウンリスト（年代を一覧から選択） |
  | B      | スライダー（タイムラインを左右にドラッグ） |
  | C      | 両方を併用（スライダー＋主要年代のボタン） |
  | D      | 入力フィールド（年を直接入力）       |

> スライダーだと年を均等な間隔で横に並べていくと思うのですが、データソース的
に選択できる年代の間隔がバラバラなので直感に反するUIになるのではないかと思っ
ています。年代のボタンを横に並べるのはどうでしょうか。

⏺ おっしゃる通りです。historical-basemapsのデータは年代の間隔が不均等（例：
  紀元前2000年→紀元前1000年→500年→1648年など）なので、等間隔のスライダーは実
  際のデータ構造と合わず、ユーザーを混乱させる可能性があります。

  年代ボタンを横に並べる方式は良いアプローチですね。利点として：

  - 選択可能な年代だけが表示されるので、「この年代は存在しない」という問題が
  起きない
  - 間隔の不均等さが視覚的に自然に表現される
  - タップ/クリックで直接選択でき、操作が明確

  追加の検討点として：

  | 方式         | 説明                          |
  |------------|-----------------------------|
  | 単純な横並びボタン  | 全年代をボタンで横に並べる（スクロール可能）
  |
  | グループ化ボタン   |
  時代区分（古代・中世・近代など）でグループ分けして表示 |
  | 横並び＋現在位置強調 | 横並びボタンで、選択中の年代を中央に表示・強調
    |

  単純な横並びボタンで進めますか？それとも何か追加の工夫がありますか？

> 時代区分は明確に定義するのが難しそうなので「横並び＋現在位置強調」で

⏺ 了解です。「年代ボタンを横並びにし、選択中の年代を中央に表示・強調する」方
  式で記録します。
