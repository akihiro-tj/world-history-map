# 機能提案イシュー起票

ユーザーストーリー形式で GitHub Issue を起票する軽量ワークフロー。
1イシュー1ストーリー。複数視点がある場合は複数回実行する。

## フロー

### 1. 情報をインタラクティブに収集する

AskUserQuestion を使って以下を対話的に聞き出す。
ユーザーの発言から推測できる項目はスキップし、足りない部分だけを聞く。

1. **誰が** — ユーザーの役割（デフォルト: 「地図の利用者」）
2. **何を** — 実現したいこと（必須）
3. **なぜ** — なぜそれが必要か（必須）

### 2. 提案・深掘りする

収集した情報をもとに、プロジェクトのコードベースや既存機能の文脈を踏まえて提案する。
2〜3個程度に絞り、AskUserQuestion で取捨選択してもらう。

提案の観点:
- ユーザーの案をより良くする改善案やアプローチの選択肢
- 一緒に検討すると良い関連機能
- ユーザーが見落としていそうな考慮点

採用された提案はメモセクションに反映する。
全て不要と判断された場合はそのまま次に進む。

### 3. メモを任意で収集する

提案以外に追加のアイデア・制約・関連情報があるか聞く。
なければ「メモ」セクションごと省略する。

### 4. イシュータイトルを生成する

機能を動詞形で簡潔に表すタイトルを作成する。
例: 「年表の表示をタイムラインUIに変更する」「領土クリック時に詳細情報を表示する」

### 5. イシュー本文を組み立てる

```markdown
## ユーザーストーリー

[誰が]として、[何を]したい。
なぜなら[なぜ]だから。

## メモ

- [収集したメモ]

---
🤖 Generated with [Claude Code](https://claude.com/claude-code)
```

メモがない場合は「## メモ」セクションごと省略する。

### 6. ユーザーに確認する

組み立てたタイトルと本文をユーザーに提示し、起票前に確認を取る。

### 7. イシューを作成する

```bash
gh issue create --title "<title>" --body "<body>" --label "proposal"
```

「proposal」ラベルが存在しない場合は先に作成する:

```bash
gh label create proposal --description "Feature proposal" --color "0E8A16"
```

### 8. 結果を報告する

作成されたイシューの URL をユーザーに表示する。
