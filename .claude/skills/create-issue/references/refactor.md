# リファクタリング提案 (refactor)

## 収集する情報

AskUserQuestion で以下を対話的に聞き出す。
ユーザーの発言から推測できる項目はスキップし、足りない部分だけを聞く。

1. **対象** — リファクタリング対象のコード・モジュール（必須）
2. **現状の問題** — 今の何が問題か（必須）
3. **改善の方向性** — どうなっていると良いか（任意、Claude が提案してもよい）

## 深掘り

対象コードを実際に読み、具体的なリファクタリング方針を提案する。
2〜3個程度に絞り、AskUserQuestion で取捨選択してもらう。

提案の観点:
- 設計パターンの適用
- 責務の分離・モジュール構成の改善
- 型安全性の向上
- テスタビリティの改善

## タイトル

「〜をリファクタリングする」「〜の責務を分離する」など、改善内容を簡潔に表す。
例: 「map-store の状態管理ロジックを hooks に分離する」

## テンプレート

```markdown
## 現状の問題

[現状の問題]

## 対象

[対象コード・モジュール]

## 改善の方向性

[改善の方向性]

## メモ

- [追加情報]

---
🤖 Generated with [Claude Code](https://claude.com/claude-code)
```

メモがない場合は「## メモ」セクションごと省略する。
