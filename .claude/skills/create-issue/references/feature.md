# 機能提案 (proposal)

## 収集する情報

AskUserQuestion で以下を対話的に聞き出す。
ユーザーの発言から推測できる項目はスキップし、足りない部分だけを聞く。

1. **誰が** — ユーザーの役割（デフォルト: 「地図の利用者」）
2. **何を** — 実現したいこと（必須）
3. **なぜ** — なぜそれが必要か（必須）
4. **解決策** — 具体的な解決策の案（任意。深掘りで補える）

## 深掘り

収集した情報をもとに、プロジェクトのコードベースや既存機能の文脈を踏まえて提案する。
2〜3個程度に絞り、AskUserQuestion で取捨選択してもらう。

提案の観点:
- ユーザーの案をより良くする改善案やアプローチの選択肢
- 一緒に検討すると良い関連機能
- ユーザーが見落としていそうな考慮点

採用された提案は「提案する解決策」セクションに反映する。
全て不要と判断された場合はそのまま次に進む。

## タイトル

機能を動詞形で簡潔に表す。
例: 「年表の表示をタイムラインUIに変更する」「領土クリック時に詳細情報を表示する」

## テンプレート

```markdown
## ユーザーストーリー

[誰が]として、[何を]したい。
なぜなら[なぜ]だから。

## 提案する解決策

1. **[解決策名]**: [説明]
2. **[解決策名]**: [説明]

## 背景

[課題の背景や文脈を説明する]

## 課題

- [現状の具体的な問題点]

## メモ

- [収集したメモ]

---
🤖 Generated with [Claude Code](https://claude.com/claude-code)
```

各セクションの省略ルール:
- 「提案する解決策」— 解決策が未定の場合は省略可
- 「背景」— ユーザーストーリーだけで文脈が十分な場合は省略可
- 「課題」— 背景と重複する場合は省略可
- 「メモ」— メモがない場合は省略
