# 調査・検討 (investigation)

## 収集する情報

AskUserQuestion で以下を対話的に聞き出す。
ユーザーの発言から推測できる項目はスキップし、足りない部分だけを聞く。

1. **調査テーマ** — 何を調べたいか（必須）
2. **背景** — なぜ今調べる必要があるか（必須）
3. **期待するアウトプット** — 調査結果をどう使うか（任意）

## 深掘り

コードベースや技術的な文脈を踏まえ、調査の切り口やスコープを提案する。
AskUserQuestion で方向性を確認する。

提案の観点:
- 調査スコープの明確化（広すぎ・狭すぎの調整）
- 関連する既存の実装・設計判断
- 検証すべき仮説やリスク

## タイトル

「〜を調査する」「〜の技術選定を検討する」など、調査内容を簡潔に表す。
例: 「タイルレンダリングの WebGL 移行を検討する」

## テンプレート

```markdown
## 調査テーマ

[調査テーマ]

## 背景

[背景]

## 期待するアウトプット

[期待するアウトプット]

## メモ

- [追加情報]

---
🤖 Generated with [Claude Code](https://claude.com/claude-code)
```

期待するアウトプットがない場合は「## 期待するアウトプット」セクションごと省略する。
メモがない場合は「## メモ」セクションごと省略する。
