# RICE スコアリング手順

## 各軸の定義と採点基準

### Reach（リーチ）

一定期間内に影響を受けるユーザー数の推定。

| スコア | 定義 |
|--------|------|
| 3 | 全ユーザーに影響する（コア体験の変更） |
| 2 | 大部分のユーザーに影響する（主要機能の改善） |
| 1 | 一部のユーザーに影響する（特定ユースケース向け） |
| 0.5 | 少数のユーザーに影響する（ニッチな機能） |

### Impact（インパクト）

個々のユーザーへの影響度。

| スコア | 定義 |
|--------|------|
| 3 | 大きなインパクト（体験が根本的に変わる） |
| 2 | 高いインパクト（体験が明確に改善される） |
| 1 | 中程度のインパクト（便利になる） |
| 0.5 | 低いインパクト（わずかに改善） |
| 0.25 | 最小限のインパクト（ほぼ気づかない） |

### Confidence（確信度）

この推定にどれだけ自信があるか。

| スコア | 定義 |
|--------|------|
| 1.0 | 高い確信（データや先行事例に基づく） |
| 0.8 | 中程度の確信（合理的な推測） |
| 0.5 | 低い確信（仮説段階） |

### Effort（工数）

実装に必要な作業量（人週単位で推定）。

コードベースからの推定方法:
- 新規コンポーネント作成: コンポーネントの複雑さから推定
- 既存コンポーネント修正: 変更範囲と影響範囲から推定
- データパイプライン変更: 処理ステップ数と検証工数から推定
- インフラ変更: 設定変更の複雑さから推定

| スコア | 定義 |
|--------|------|
| 0.5 | 数時間の作業 |
| 1 | 1 週間程度の作業 |
| 2 | 2 週間程度の作業 |
| 3 | 1 ヶ月程度の作業 |
| 5 | 1 ヶ月以上の作業 |

## スコア計算

```
RICE Score = (Reach × Impact × Confidence) / Effort
```

## 出力フォーマット

```markdown
| 機会 | Reach | Impact | Confidence | Effort | RICE Score |
|------|-------|--------|------------|--------|------------|
| ... | 0.5-3 | 0.25-3 | 0.5-1.0 | 0.5-5 | 計算値 |
```

スコア降順でソートし、上位の機会を優先候補として提示する。
