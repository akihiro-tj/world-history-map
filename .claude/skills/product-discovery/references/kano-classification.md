# Kano モデル分類基準

## 3 つの品質カテゴリ

### Must-be（当たり前品質）

あって当然、なければ不満。充足しても満足度は上がらない。

判断基準:
- 競合プロダクトではほぼ標準装備されている
- ないと「壊れている」と感じられる
- ユーザーが明示的に要求しないが、なければ離脱する

例（歴史地図）: 地図のパン/ズーム、年代の切り替え、領土の可視化

### One-dimensional（一元的品質）

あればあるほど満足度が上がり、なければ不満。

判断基準:
- ユーザーが明示的に「もっと欲しい」と言える
- 競合との差別化に直結する
- 投資に対して線形的にリターンがある

例（歴史地図）: データの年代カバレッジ、領土情報の詳細度、地図の表示速度

### Attractive（魅力的品質）

あれば大きく満足度が上がるが、なくても不満にならない。

判断基準:
- ユーザーが事前に想像できない
- 「おっ」という驚きや喜びを生む
- 時間経過とともに一元的品質に移行しうる

例（歴史地図）: アニメーション遷移、タイムラプス再生、関連する歴史イベントの表示

## ライフサイクル移行

品質カテゴリは時間とともに移行する:

```
Attractive → One-dimensional → Must-be
```

かつては魅力的だった機能も、業界標準になれば当たり前品質になる。
分類時は現時点での市場標準を基準にする。

## 投資戦略への示唆

| カテゴリ | 投資判断 |
|----------|---------|
| Must-be | 欠如していれば最優先で対応。ただし過剰投資は避ける |
| One-dimensional | 競合優位性に直結するため、戦略的に投資する |
| Attractive | 差別化の源泉。リスクは高いがリターンも大きい |
