# コードベース分析チェックリスト

## 1. ユーザーフロー分析

- [ ] ルーティング構造を確認し、ユーザーが辿れるパスを列挙する
- [ ] エントリーポイント（初回訪問時の体験）を確認する
- [ ] 主要なユーザーアクション（クリック、入力、ナビゲーション）を列挙する
- [ ] デッドエンド（行き止まり）になるフローがないか確認する
- [ ] 状態管理を確認し、ユーザーの文脈がどう保持されるか把握する

## 2. UI コンポーネント分析

- [ ] コンポーネントツリーを走査し、UI の構成を把握する
- [ ] インタラクティブ要素（ボタン、スライダー、入力欄等）を列挙する
- [ ] レスポンシブ対応の状況を確認する
- [ ] アクセシビリティ対応（ARIA 属性、キーボード操作等）を確認する
- [ ] 表示状態のバリエーション（ローディング、エラー、空状態）を確認する

## 3. データフロー分析

- [ ] 外部データソース（API、ファイル、DB）を列挙する
- [ ] データの取得タイミング（初期ロード、遅延ロード、オンデマンド）を確認する
- [ ] キャッシュ戦略を確認する
- [ ] データの鮮度・更新頻度を確認する
- [ ] データ量とスケーラビリティを評価する

## 4. パフォーマンス分析

- [ ] バンドルサイズと分割戦略を確認する
- [ ] 重い処理（地図レンダリング、大量データ処理等）を特定する
- [ ] ネットワークリクエストの数と頻度を確認する
- [ ] 画像・アセットの最適化状況を確認する

## 5. 技術的負債・DX 分析

- [ ] テストカバレッジと品質を確認する
- [ ] 型安全性の状況を確認する
- [ ] 依存パッケージの更新状況を確認する
- [ ] ビルド・デプロイの効率性を確認する
- [ ] コードの重複や非効率なパターンを特定する

## 6. 既存仕様との差分分析

- [ ] `specs/` 配下の仕様ドキュメントを確認する
- [ ] 仕様に書かれているが未実装の機能を特定する
- [ ] 実装済みだが仕様と乖離している箇所を特定する
- [ ] GitHub Issues で報告されている既知の問題を確認する

## 分析結果の記録

各チェック項目について、以下を記録する:
- **発見事項**: 何が分かったか
- **機会**: 改善の余地があるか
- **関連ファイル**: 対象のソースコード
