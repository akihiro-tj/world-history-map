# overview.md テンプレート

`docs/overview.md` は以下の構成に従う。

## 共通ルール

- バージョン番号、コマンド一覧、ファイルの網羅的リストは書かない
- 図は ASCII アートのみ。Mermaid は使わない
- 1 図あたりノード 7 個以内。描けないほど複雑なら図を分割するか、情報を減らす
- 表は使わない。箇条書きか短い散文で書く
- 全体で 150 行以内を目安とする

## 記述の指針

- 「何がどう流れるか」「何が何に変わるか」を中心に書く
- 個々のファイルの実装詳細には踏み込まない
- 型定義はフィールド名だけ列挙する（型アノテーション不要）
- 「よくある問い」は実際にコードを触る場面を想定する

---

## テンプレート構成

```markdown
# コードベース概要

> Last synced: YYYY-MM-DD

## このアプリは何か

1〜2 文でプロダクトの概要。

## アプリ構成

ASCII アートで 3 アプリの関係を示す。ノード 7 個以内。

## pipeline: データの流れ

系統ごとに処理の流れを ASCII アートで示す。
各ステージは 1 行の説明のみ。

## frontend: 状態設計

状態の 3 層構造を ASCII アートで示す。
各層の主要な状態を箇条書き。

## 中核の型

pipeline / frontend 間で共有される型をフィールド名のみで列挙。

## データファイル（public/data/）

実行時に参照される JSON ファイルの役割を箇条書き。

## worker: タイル配信

Worker の役割を 3〜5 行で説明。

## よくある問い

実際の開発シナリオに基づく Q&A。
各問いは「操作 → 結果」の流れを箇条書きで示す。
```
