# Worktree セットアップ

Git worktree の作成から開発環境の準備までを行う。

## 引数

- イシュー番号（例: `159`）またはワークツリー名（例: `my-feature`）

イシュー番号が渡された場合、worktree 名は `issue-<番号>` になる。

## フロー

### 1. Worktree を作成する

`EnterWorktree` ツールを使って worktree を作成する。

```
EnterWorktree(name: "issue-159")
```

### 2. 依存関係をインストールする

プロジェクトのパッケージマネージャでインストールコマンドを実行する。

### 3. イシューのコンテキストをロードする（イシュー番号が渡された場合）

```bash
gh issue view <番号> --json title,body,labels
```

取得したイシューの内容を把握し、以降の作業に備える。

### 4. 準備完了を報告する

以下の情報を表示する:

- worktree のパスとブランチ名
- イシューのタイトル（該当する場合）
