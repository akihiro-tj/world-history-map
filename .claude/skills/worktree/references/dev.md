# Worktree 開発

GitHub イシューの内容を仕様として扱い、実装から PR 作成までを実行する。

## 引数

- イシュー番号（例: `159`）

## フロー

### 1. イシュー情報を取得する

```bash
gh issue view <番号> --json title,body,labels
```

イシュー本文を仕様として読み解く。フォーマットはイシューによって異なるため、
本文全体から「何を」「どう変更するか」「完了条件は何か」を理解する。

先行イシューへの言及（例: `#158 の後に実施`）がある場合、
そのイシューがクローズ済みか確認する。未クローズならユーザーに警告し続行するか確認する。

### 2. 現状を把握する

イシュー本文で言及されているファイルやモジュールを読み込み、現状のコードを理解する。
言及がない場合はイシューの内容からコードベースを探索して関連ファイルを特定する。

### 3. 実装する

イシューの仕様に基づいて実装を進める。

### 4. 検証する

プロジェクトの検証コマンドを実行する。
失敗した場合は修正して再実行する。3回失敗したらユーザーに相談する。

### 5. コミットする

変更を適切な粒度でコミットする。

### 6. PR を作成する

`closes #<番号>` でイシューをリンクした PR を作成する。

### 7. 結果を報告する

作成された PR の URL と変更の概要を表示する。
