# フィールド定義

## Priority

Product Vision（CLAUDE.md）に基づく判断基準。

| 値 | 基準 | 例 |
|----|------|-----|
| P0: Critical | プロダクトが使えない・データ破損 | マップが表示されない、タイルが読み込めない |
| P1: High | Product Vision の3課題を直接解決する機能 | 年代スライダーによる横のつながり可視化、領土クリックで縦のつながり表示 |
| P2: Medium | UX 改善で利用体験を向上させる | ズームレベル最適化、モバイル対応改善 |
| P3: Low | コア価値への影響が小さい・nice-to-have | UI の微調整、開発者体験の改善 |

### Product Vision の3課題（参考）

1. **縦のつながり**: 特定領土の歴史を連続的に追跡できる
2. **横のつながり**: 任意の年の世界全体を一画面で把握できる
3. **地理的知識**: 地理情報を視覚的・直感的に学べる

## Size

作業規模の見積もり基準。

| 値 | 目安時間 | スコープ |
|----|----------|----------|
| XS | 数時間 | 1-2 ファイルの修正 |
| S | 1日 | 数ファイルの修正、小規模な機能追加 |
| M | 2-3日 | 新コンポーネントの作成、中規模な機能 |
| L | 1週間 | 新サブシステム、複数コンポーネントにまたがる変更 |
| XL | 1週間超 | アーキテクチャ変更、大規模リファクタリング |

## Status

GitHub Projects のデフォルト + カスタム。

| 値 | 用途 |
|----|------|
| Backlog | 整理済みだが着手予定なし |
| Todo | 次に着手する候補 |
| In Progress | 作業中（WIP 制限あり） |
| Done | 完了 |
