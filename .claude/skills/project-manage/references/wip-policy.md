# WIP 制限ポリシー

個人開発におけるフォーカスを維持するための制限。

## 制限値

| Status | 上限 | 備考 |
|--------|------|------|
| In Progress | 2件（推奨1件） | 並行作業はコンテキストスイッチコストが高い |
| Todo | 5件 | 多すぎる Todo は意思決定疲れを招く |
| Backlog | 制限なし | 整理済みであれば問題ない |

## 超過時の対処

### In Progress が上限超過の場合

1. 完了できるものがないか確認する
2. ブロックされているものがあれば Backlog に戻す
3. 新規着手は既存作業の完了後にする

### Todo が上限超過の場合

1. Priority の低いものを Backlog に戻す
2. 着手時期が不明なものを Backlog に戻す
